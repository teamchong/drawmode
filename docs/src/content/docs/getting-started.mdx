## Installation

drawmode runs as an MCP server. No separate install is needed — `npx` handles it.

```bash
# Claude Code / Cursor (stdio mode)
npx drawmode --stdio

# HTTP mode (Streamable HTTP on port 3001)
npx drawmode
```

## Client Configuration

### MCP Client Configuration

Add this to your MCP client's configuration file:

- **Claude Code**: `~/.claude/settings.json` or project `.mcp.json`
- **Claude Desktop (macOS)**: `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Claude Desktop (Windows)**: `%APPDATA%\Claude\claude_desktop_config.json`
- **Cursor**: Cursor MCP settings

```json
{
  "mcpServers": {
    "drawmode": {
      "command": "npx",
      "args": ["drawmode", "--stdio"]
    }
  }
}
```

## Your First Diagram

Once configured, ask your LLM to draw a diagram. The LLM receives a single `draw` tool and writes TypeScript code against the Diagram SDK:

```typescript
const d = new Diagram();
const web = d.addBox("Web App", { row: 0, col: 0, color: "frontend" });
const api = d.addBox("API Server", { row: 0, col: 1, color: "backend" });
const db = d.addBox("Database", { row: 1, col: 1, color: "database" });
d.connect(web, api, "REST");
d.connect(api, db, "queries");
return d.render({ format: "url" });
```

This produces a shareable [excalidraw.com](https://excalidraw.com) URL printed in the tool response. Click it to view and edit the diagram in your browser. For local files, use `format: "excalidraw"` — this writes a `.excalidraw` file you can open directly in VS Code (with the Excalidraw extension) or drag into excalidraw.com.

## Output Formats

| Format | Description | Works in |
|--------|-------------|----------|
| `excalidraw` | `.excalidraw` JSON file | Claude Code, Cursor, VS Code |
| `url` | Shareable excalidraw.com link | All clients |
| `png` | PNG image (requires puppeteer) | Local with puppeteer, Cloudflare Worker |
| `svg` | SVG markup | Local with puppeteer |

## How It Works

1. LLM receives one tool (`draw`) with TypeScript type definitions
2. LLM writes code against the `Diagram` SDK
3. Executor runs the code — SDK handles labels, colors, IDs
4. Graphviz `dot` engine handles layout positioning and edge routing
5. Zig WASM handles validation
6. Output: `.excalidraw` file, excalidraw.com URL, PNG, or SVG
